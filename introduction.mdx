## What is Bad UI?

Bad UI is a curated, research-backed catalog of UI antipatterns — the recurring design and accessibility failures that cost products real users, real revenue, and real trust.

Every pattern in the library is documented with:

- **What went wrong** — a clear description of the broken pattern and its observable behavior
- **Why it fails** — the cognitive, accessibility, or design principle being violated
- **Business impact** — quantified conversion damage, abandonment rates, and support costs
- **Standards mapping** — WCAG 2.1/2.2 success criteria and Nielsen's 10 usability heuristics
- **The fix** — production-ready React/Tailwind CSS code you can drop into your codebase

## Why this exists

Most teams know their UI has problems. The hard part is communicating *why* those problems matter — to stakeholders, to engineers who built them, and to designers who approved them.

Bad UI provides the evidence layer: real statistics, accessibility standards citations, and conversion impact data that transform vague complaints into defensible, actionable engineering priorities.

## Impact severity levels

Each antipattern is assigned one of four impact levels based on observed user abandonment, error rates, and accessibility failure severity:

| Level | Description |
|-------|-------------|
| **Critical** | Severe usability roadblock. High risk of immediate abandonment. |
| **High** | Major friction point. Significantly degrades user satisfaction. |
| **Medium** | Consistent annoyance. Slows down users and erodes trust. |
| **Low** | Minor polish issue. Mostly aesthetic or edge-case friction. |

## Tech stack

Bad UI is built with Next.js 16, React 19, Tailwind CSS 4, and Supabase. The antipattern database is a PostgreSQL table with row-level security, served through Supabase's JavaScript client.
