## Category reference

| Category | Slug | Key antipattern types |
|---|---|---|
| Forms | `forms` | Late validation, hidden requirements, input-clearing blur handlers, inaccessible date pickers |
| Navigation | `navigation` | Scroll hijacking, infinite-scroll footer traps, hamburger menus, broken breadcrumbs |
| Accessibility | `accessibility` | Insufficient contrast, missing ARIA labels, focus traps, keyboard-hostile controls |
| Responsive & Mobile | `mobile` | Undersized tap targets, horizontal overflow, illegible font sizes, pinch-zoom disabled |
| Feedback & States | `feedback` | Swapped button styles, ephemeral error toasts, silent success, missing loading indicators |
| Layout & Spacing | `layout` | Flat hierarchy, misaligned elements, information overload, inconsistent whitespace |
| Typography | `typography` | Sub-12 px body text, excessive line density, orphaned words, poor typeface pairing |
| Dark Patterns | `dark-patterns` | Cancellation labyrinths, invisible close buttons, confirmshaming, hidden costs |
| Loading & Timeouts | `loading` | Silent timeouts, unsaved state after refresh, unnecessary splash screens, no skeleton UI |
| Control Theatre | `control-theatre` | Blocked paste, disabled right-click, forced app installs, no keyboard shortcuts |
| Notification Spam | `notifications` | Immediate permission requests, pre-engagement popups, notification fatigue loops |
| Broken Micro-UX | `micro-ux` | Indicatorless carousels, scroll-jumping accordions, hover traps, focus loss on update |

## How patterns are rated

Every antipattern carries one of four impact levels derived from observed abandonment rates, error rates, and accessibility failure severity.

| Level | Description |
|---|---|
| **Critical** | Severe usability roadblock. High risk of immediate abandonment. |
| **High** | Major friction point. Significantly degrades user satisfaction. |
| **Medium** | Consistent annoyance. Slows down users and erodes trust. |
| **Low** | Minor polish issue. Mostly aesthetic or edge-case friction. |

## Standards coverage

Patterns are cross-referenced against two authoritative frameworks:

- **WCAG 2.1 / 2.2** — Web Content Accessibility Guidelines success criteria (e.g., 1.4.3 Contrast Minimum, 2.4.7 Focus Visible)
- **Nielsen's 10 Usability Heuristics** — Jakob Nielsen's foundational heuristics (e.g., #1 Visibility of system status, #3 User control and freedom)

<Tip>
  Start with the category most relevant to your current sprint. If you're auditing a checkout flow, begin with **Forms** and **Dark Patterns**. If you're addressing an accessibility review, begin with **Accessibility** and **Typography**.
</Tip>
