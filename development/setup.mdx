## Prerequisites

Before you begin, make sure you have the following installed:

| Tool | Minimum version | Notes |
|------|-----------------|-------|
| **Node.js** | 20.x LTS | Required by Next.js 16 |
| **npm** | 10.x | Bundled with Node.js; yarn, pnpm, or bun also work |
| **Supabase CLI** | Latest | Required to run local migrations |
| **Git** | Any recent | For cloning the repository |

<Tip>
Install the Supabase CLI with `npm install -g supabase` or via the [official install guide](https://supabase.com/docs/guides/cli/getting-started).
</Tip>

## Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/jaanque/badui.git
    cd badui
    ```
  </Step>

  <Step title="Install dependencies">
    <CodeGroup>

    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```

    ```bash bun
    bun install
    ```

    </CodeGroup>

    The main runtime dependencies installed are:

    - `next` 16.1.6 and `react` / `react-dom` 19.2.3
    - `@supabase/ssr` ^0.9.0 ‚Äî cookie-based session management for Server Components
    - `@supabase/supabase-js` ^2.99.1 ‚Äî Supabase query client
    - `@base-ui/react` ^1.3.0 and `@radix-ui/react-slot` ^1.2.4 ‚Äî headless UI primitives
    - `tailwindcss` ^4 with `tw-animate-css` and `tailwind-merge`
    - `lucide-react` ^0.577.0 ‚Äî icon library
  </Step>

  <Step title="Configure environment variables">
    Create a `.env.local` file in the project root. The `.gitignore` excludes all `.env*` files so this will never be committed.

    ```bash .env.local
    NEXT_PUBLIC_SUPABASE_URL=https://<your-project-ref>.supabase.co
    NEXT_PUBLIC_SUPABASE_ANON_KEY=<your-anon-public-key>
    ```

    Both variables are **required** at runtime and at build time. You can find them in your Supabase project dashboard under **Settings ‚Üí API**.

    <Warning>
    These variables are prefixed with `NEXT_PUBLIC_` and are therefore exposed in the browser bundle. They must use only the anon (public) key ‚Äî never a service role key.
    </Warning>
  </Step>

  <Step title="Apply database migrations">
    See the [database setup guide](/development/database) for instructions on creating your Supabase project and running the three migration files in order.
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    Open [http://localhost:3000](http://localhost:3000) in your browser. The Next.js dev server supports hot module replacement ‚Äî changes to files in `app/` and `components/` are reflected instantly without a full reload.
  </Step>
</Steps>

## Available scripts

All scripts are defined in `package.json`:

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `next dev` | Start the local development server with HMR |
| `build` | `next build` | Produce an optimized production build |
| `start` | `next start` | Serve a production build locally |
| `lint` | `eslint` | Run ESLint across the project (config in `eslint.config.mjs`) |

```bash
npm run dev      # development
npm run build    # production build
npm run start    # serve production build
npm run lint     # lint check
```

### Path aliases

The `@/*` alias maps to the project root, configured in `tsconfig.json`:

```ts
import { createClient } from "@/lib/supabase/server";
import type { DbAntipattern } from "@/components/home/data";
```

### Middleware

`middleware.ts` runs on every request except static files and images. It uses `createServerClient` from `@supabase/ssr` to refresh the user session and propagate updated cookies via `NextResponse`:

```ts middleware.ts
export const config = {
  matcher: [
    "/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)",
  ],
};
```

<Note>
The middleware calls `supabase.auth.getUser()` on every matched request. This is required to keep the Supabase session cookie current ‚Äî do not remove this call.
</Note>
